Example: Jingdong Price Scraper

This section briefly shows how to scrape the real-time price of a product from Jingdong, and list some issues with solutions. We choose product with ID 100003559548 as our example.

Parse HTML to Extract the Price

Instinctively, we can see the price while visiting 100003559548, so why don’t we extract the price from HTML directly?

  • For Chrome users, open 100003559548, and right-click context, and click Inspect.
  • Click Element and search the price, say 188.00. Then we can see the HTML piece like below.
  • Actually the price 188.00 is wrapped in the tag span, and if we try to parse this tag, we get nothing.

JS Query Request

TBH, the price on 100003559548 is queried by a JS request. To parse this request, search some prices form this website (probably not 188.00, tried, but not useful).

  • Click Network and search 192.00.
  • Analyze items in the left result box, and click Headers. Here we can find a request URL.
  • Analyze this URL and we may guess the JS query request’s format as below.
  • Replace ID by 2622752 and visit Price Page. Congratulations, we get the magic number 188.00.
  • The HTML content is pretty simple and obvious.

Header Simulation

Visiting Price Page from a browser and python are so different. If we try to visit it directly as below.

Then, we will be blocked by their robot detection system and receive this response.

The reason is that the request sent by urllib contains a robotic header, which leads to the block. To overcome this, we can simulate a request header sent by human.

  • Inspect Price Page.
  • Click Network, and select the only request, and click Headers.
  • Copy the header and pass it to request.Request().
